Enumerable method: group_by

Technical Blog #4

June 14th, 2015

Imagine a scenario in which you are having twins. Your wife and you agree on most things, but have always had different ideas on what constitutes a good baby name -- you have always been partial to the short, 3 letter names, while your wife has always felt that 8 character names were better. You have agreed that each of you will get to name one of the babies.

Your friend at the US Census Bureau has sent you an array, called names that he exctracted from a database of all of the existing names in America. You would like to organize this collection based on name length so that you can easily see all the names that are 3 letters long, and your wife can see all of the names that are 8 letters long. Luckily for you, the group_by method can be used on your enumerable!

The group_by method works by acting on a collection object (in this case an array) and returning a hash. The key of this hash will be the result of whatever code block is specified after group_by, and the value of this hash will be an array of all of the original items of the collection object that share the same resultant key. This is best explained by looking at a group_by call on your names array.

names.group_by {|name| name.length}

would return the following (hypotehtical) hash:

{2 => ["Bo", "Jo",...], 3 => ["Boa", "Jon"...], 8 => ["Jonathan", "Philippo"...}

group_by works by going through each item in the collection and performing the code block on all of them -- in this case finding their length. The length is now the key in the hash, and the value is an array of all the names from the original array that are of that length.

Now that you have organized these names into a hash based on name length, you can easily see the three-letter names grouped together and your wife can easily see all of the eight-letter names grouped together.
