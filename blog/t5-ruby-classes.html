Ruby Classes

Technical Blog #5

June 17th, 2015

Classes in Ruby can be seen as a way to organize many objects that share similar behaviors and characteristics. In order to understand classes and why they are useful, we will first look at some classes that are built in to Ruby, and then we will create our own class.

Built-in Classes

If you have had any basic experience with Ruby, you have already dealt with classes, whether you know it or not. String, Integer, Array, and Hash are all examples of classes that have been built-in to Ruby. We'll take a look at the String class to help us understand what a class is.

You have probably already created a string variable before, something along the lines of my_name = "Travis". Ruby is smart enough to figure out from those quotation marks that the my_name variable represents a string. A more precise description of what is happening is that the my_name variable being created is an instance of the String class. The advantage of this is that, simply by creating a new string object, you can automatically do many methods on this new object that are already built in to the String class. All you have to do to witness this is call the methods method on your new variable. You should get something like this:

Creating Our Own Class

To make classes a little less abstract, let's come up with a "Real World" example of how a class might work. In this example, we will make a class for "Athletes"

If you think about it, athletes share many fundamental characteristics. They have a name, an age, a team, a salary, and a professional sport that they play. While these characteristics vary from one athlete to the next, we still tend to group athletes together in the same category. They also share common behaviors -- I think it's safe to say that all athletes exercise. Given these shared characteristics and behaviors, it makes sense to create a class for athletes.

To create the athlete class, we would do something like this: 

class Athlete
end

When we create a new athlete (an instance of the class Athlete), we want to be able to specify some of their characteristics. We are aided by the initialize method that is a special method that can be defined for your class. We will define it with the characteristics we mentioned above as arguments. Take a look at the following code:

class Athlete

def initialize(name, age, team, salary,sport)
  @name = name
  @age = age
  @team = team
  @salary = salary
  @sport = sport
end

end

Much of this may sound foreign, but look what it allows you to do when you create a new athlete, using the following syntax:

kobe = Athlete.new("Kobe Bryant", 36, "Los Angeles Lakers", "23 million")

You have now created a new instance of the Athlete class called kobe, and endowed him with certain attributes. The beauty of this is that you can create as many instances of the Athlete class as you want, such as for Peyton Manning, Tiger Woods, Derek Jeter, etc., and endow them all with their specific attributes.

We also said that, in addition to sharing certain characteristics, athletes also share certain behaviors. Let's extend this to our Athlete class, using the updated code below:

class Athlete

def initialize(name, age, team, salary,sport)
  @name = name
  @age = age
  @team = team
  @salary = salary
  @sport = sport
end

def exercise
  puts "#{name} loves to exercise"
end

end

We have now created an exercise method that you can call on any instance of the Athlete class. You can now type kobe.exercise and expect to get the following output: 

=>Kobe Bryant loves to exercise

While this example is a very simple class (and not all that practical in actual programming), I hope it indicates how classes can help organize similar objects with similar attributes and behaviors!
