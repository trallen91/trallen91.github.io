The Differences between Relative, Absolute, and Fixed Position

Technical Blog #2

June 1st, 2015

One of the advantages CSS offers is its ability to position different html elements on the web page. This blog aims to explain the four different position methods in CSS.

Static Positioning

The default position of HTML elements is the "static" position. This means that the element will be positioned according to the flow of the page. If one sets the position to "static" or does not specify a position for an element, then this element will not be affected by the "top", "bottom", "left", or "right" properties.

Fixed Positioning

If an element is given a "fixed" position, then this means that the element will be positioned relative to the browser window . This means that even if you scroll through the window, that element will remain in a fixed position.

Whereas elements in a static position are "immune" to the effects of top, right, bottom, and left properties, fixed elements can be positioned on the browser window using these properties.

When an element is given a fixed position, it is removed from the normal flow. This means that the other elements behave like the fixed positioned element does not exist. As a result, these fixed positioned elements can overlap other elements.

Relative Positioning

A relative positioned element is positioned relative to its normal position. This allows you to use the top, right, bottom, and left properties to position the element relative to where it would have fallen in the normal flow (i.e., had position not been set or been set to static).

As will be better understood after the following section, elements are sometimes given the relative position just so that it can be used a placeholder for a child element that is given the "absolute" position.

Absolute Positioning

An element given the absolute position is positioned relative to the nearest "positioned" ancestor. A "positioned" element is defined as an element that has any position property besides "static".

If an element with an absolute position does not have any positioned ancestors, it uses the document body as its reference point

Margins, Borders, and Padding

Bonus Technical Blog #1 for Week 2

June 1st, 2015

The CSS Box Model is a design model that treats all HTML elements to be "boxes." This box consists of (from most central to most peripheral):

  • Content: Where the text and images appear
  • Padding: Refers to the area surrounding the content (i.e. the area between the content and the border).
  • Border: This surrounds the padding and the content.
  • Margin: This is the area outside the border.

The Display Properties: Inline vs Inline-Block vs. Block

Bonus Technical Blog #2 for Week 2

June 1st, 2015

In the blog post above, we explained how each HTML element is a rectangular box. These rectangular boxes have "display" properties that will determine how they interact with each other. We will go through the major display property values.

Inline

The default value for elements is "inline". An inline element can wrap some text inside a paragraph without disrupting the flow of that paragraph.

Inline elements can accept margin and padding properties, but they will ignore the "top" and "bottom" aspects of these properties. They will also ignore width and height properties entirely.

The common examples of the inline elements are:

  • Span
  • em
  • b

Inline-Block

An element set to inline-block is like inline in the sense that it is set "inline" with the natural flow of the text. The major difference is that you can use the width and height properties

Block

Block elements start on a new line automatically (and are therefore not "inline") and stretch out the left and right as far as it can.

There are a couple block-level elements that are automatically set to block:

  • div
  • p
  • header
  • footer
  • section
